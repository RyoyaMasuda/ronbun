Part 0 — TL;DR / はじめに

【論文×実装】Agent Hospital — SEAL/MedAgent-Zeroの解説、小規模再現とRAGチャットへの適応

本ページは Part 0（0. TL;DR / 1. はじめに）です。Part 1 以降は別ファイルで執筆します。

0. TL;DR

  • Agent Hospital は、患者・医師・看護師がすべて LLM エージェントである仮想病院（シミュラクラ）。診察〜治療をシミュレーションし、医師エージェントが経験を積んで「進化」する論文（清華大学 AIR 等, arXiv:2405.02957）。
  • 進化の中核は MedAgent-Zero。症例ラベル付きデータでファインチューニングするのではなく、凍結した LLM に 反省ループ + 経験ベース（RAG） を載せて強くする。
  • 仮想病院で約数万人の患者を診た後、MedQA（USMLE 形式）で既存手法を上回る精度（GPT-4o ベースで 92% 超）を報告している。
  • 本記事では、(1) 論文の解説、(2) ミニ版の再現、(3) 再現で得た知見を病院向け RAG チャットアプリにどう活かせるか、の3つを扱う。

1. はじめに

1-1. この記事でやること

現在、病院向けの RAG チャットアプリ（院内マニュアル等を検索して回答する構成）を開発しています。院内資料の Q&A には RAG がよく合うと感じており、そのうえでワークフロー制御やツール利用などエージェント的な拡張も調べています。

その調査の一環で、医療ドメインのマルチエージェント論文 Agent Hospital に出会い、次の3段階で整理することにしました。

段階内容
論文解説SEAL / MedAgent-Zero を中心に、論文をエンジニア視点で読み解く
小規模再現Agent Hospital のミニ版を構築し、動かしてみる
RAG チャットへの適応再現で得た設計パターンを、現行の RAG チャットにどう転用できるか考える

「RAG チャットだから全部 RAG で解決する」必要はありません。Agent Hospital も RAG・シミュレーション・反省・グラフ制御が組み合わさっており、RAG を核にしつつ別レイヤーを足せる好例だと捉えています。

本記事は製品の宣伝ではなく、実務の延長での論文読解と再現ログです。サービス名・クライアント名は出しません。

1-2. 対象読者

  • LLM / マルチエージェントに興味があるデータサイエンティストやエンジニア
  • 小規模に再現してみたい人、内容解説だけでなく実装例もほしい人
  • RAG / チャットボットを開発している人

1-3. 記事の構成

記事全体は次の Part に分かれています（本ファイルは Part 0 のみ）。

Part 0 … 0. TL;DR / 1. はじめに          ← 本ファイル

Part 2 … 8〜10章：ミニ版の設計・実装・結果

Part 1（論文解説） では、背景・SEAL・MedAgent-Zero・実験結果・読んで残った疑問を扱います。MedQA による「仮想→現実」のアライメントは論文側の話として 6 章で解説します。

Part 2（小規模再現） では、論文の一部を省略したミニ版 Agent Hospital を AWS Strands 等で構築し、進化前後の挙動を確認します。

Part 3（RAG チャットへの適応） では、Part 3 の「適応」は MedQA の全量再現や実臨床投入ではありません。反省ループ・経験蓄積・ワークフロー分離など、院内資料 Q&A 向け RAG チャットに持ち帰れるパターンを整理します。

動機や開発背景の詳細は 2 章、適用の具体案は 11 章 で述べます。Part 0 では全体像だけ押さえておいてください。
