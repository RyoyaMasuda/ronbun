================================================================================
【設計仕様書】AgentHospital（自己進化エージェント）のStrands実装方針
================================================================================

1. 全体コンセプトとコア技術
--------------------------------------------------------------------------------
* 目的: 
  AgentHospitalの論文に倣い、診察の経験（成功・失敗）を記憶し、自己進化する
  マルチエージェント・シミュレーションを構築する。
* フレームワーク: 
  AWS Strands（軽量でモデル駆動なエージェントSDK）。
  LangChain/LangGraphなどの重量級フレームワークを避け、ブラックボックス化を防ぐ。
* 状態遷移（ワークフロー）: 
  Strandsの `GraphBuilder` を使用し、Pythonの確定的な関数とLLMエージェントを
  組み合わせた有向グラフ（Directed Graph）を構築する。

2. 記憶（Memory）の設計方針
--------------------------------------------------------------------------------
重厚なベクトルDB（一般的なNaive RAG）は採用しない。
代わりに、「ディレクトリとファイル」を記憶の保管庫（ストレージ）として直接扱い、
LLMが「ツール（Skills）」として自律的に読み書きする「エージェント型RAG」を採用する。

* 記憶の保存先（ストレージ構造）:
  ├── storage/
  │   ├── doctor/
  │   │   └── global_lessons.json  # 過去の全患者から得た知見や反省を蓄積する共通ノート
  │   └── patients/
  │       ├── patient_001.json     # 各患者の個別の診察カルテ（履歴）
  │       └── patient_002.json
  
* 記憶の更新戦略:
  * 完全上書き（物理削除）はしない。
  * 「過去の間違い」もエージェントの成長記録（メタ認知の材料）として残す。
  * 古い記憶には無効フラグ（is_active = False / 論理削除）を立て、
    新しい修正された記憶を追記（Append-only）する形式をとる。

3. スキル（Skills / Tools）の構成
--------------------------------------------------------------------------------
エージェント（またはグラフのノード）が特定のタイミングで実行する確定的なPythonの機能群。

* `read_global_lessons`: 
  診察開始時に共通の知見ノート（global_lessons.json）を読み込み、プロンプトに展開する。
* `write_patient_record`: 
  診察の経過や結果を個別の患者ファイル（patient_xxx.json）に書き込む。
* `update_global_lessons`: 
  診察後の反省フェーズで、新しい教訓を共通ノート（global_lessons.json）に追記・修正する。
  （古いエントリを無効化し、新しいエントリをリンクして追記）

4. グラフ（ステートマシン）のワークフロー
--------------------------------------------------------------------------------
StrandsのGraphBuilderを使用して、以下のノードとエッジ（遷移）を構築する。

* ノード一覧:
  1. 【Pre-Consult】(種類: Python関数)
     診察前処理。`read_global_lessons` を実行し、過去の知見をシステムにロードする。
  2. 【Interview】(種類: LLMエージェント)
     診察中。患者と対話し、必要に応じて検査スキル（血液検査、X線など）を実行する。
  3. 【Patient/Env】(種類: Python関数 or LLMエージェント)
     患者役。医師（Interviewノード）からの質問や検査指示に対して症状や結果を返す。
  4. 【Reflection】(種類: Python関数 + LLM推論)
     反省。診察終了後、正解病名と自身の診断を比較。
     なぜ間違えたかを分析し、`update_global_lessons` を実行して知見をアップデートする。

* ループ制御（条件付きエッジ / Conditional Edges）:
  * `Interview` ノードの出力結果（LLMのレスポンス）を判定するルーター関数を挟む。
  * LLMが「まだ問診が必要」と判断している場合は、`Patient/Env` ノードへ遷移（会話ループ）。
  * LLMが「最終診断（Diagnosis）を下した」と判定した瞬間、ループを抜けて `Reflection` ノードへ遷移させる。
  * これにより、whileループを自分でゴリゴリ書く「だるさ」を解消しつつ、確実なフロー制御を行う。
