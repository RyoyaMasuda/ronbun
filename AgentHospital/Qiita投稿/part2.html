8. ミニ版 Agent Hospital の設計

論文を丸ごと再現するのではなく、診察 → 正誤判定 → 反省 → 経験蓄積 のコアだけを切り出します。

8-1. 再現する部分 / 省略する部分

論文の要素ミニ版
問診・診断・反省・経験蓄積再現
患者・正誤判定簡略再現（症例 JSON + ルール）
トリアージ・看護師・32 診療科・患者自動生成省略
embedding 検索省略 → ツールでファイル読み書き

8-2. スコープとワークフロー

1 回の診察が完結するフローに絞ります。実装では次の 4 ノード（処理の塊）に分けます。英語名はコード上の識別子です。

ノード名意味種類
Pre-Consult診察前準備 — 過去の教訓を読み込むPython
Interview問診・診断LLM
Patient/Env患者・環境 — 症状・検査結果を返すPython or LLM
Reflection反省 — 正解比較・教訓追記Python + LLM
 
診察前準備 → 問診・診断 ⇄ 患者・環境 → 反省
(Pre-Consult)  (Interview)  (Patient/Env)  (Reflection)

制御は AWS Strands の GraphBuilder に任せます。

8-3. 技術スタック

  • Strands + GraphBuilder — ワークフロー明示
  • 記憶 — storage/ + Agent Skills（SKILL.md + scripts/）
  • 正誤判定 — 症例 JSON の ground_truth と照合

9. 実装

9-1. 各ノードの処理

  1. Pre-Consult — global-lessons Skill で教訓読み込み
  2. Interview — 問診・診断。patient-record Skill でカルテ追記
  3. Patient/Env — 症例 JSON から応答
  4. Reflection — global-lessons Skill で教訓追記

9-2. 記憶を Skills 形式で置く

データ（storage/）と読み書きの手順（skills/）を分けます。各 Skill は SKILL.md（YAML + 本文）+ scripts/。起動時は name / description のみプロンプトに載り、必要時に有効化します。

skills/
├── global-lessons/SKILL.md + scripts/
└── patient-record/SKILL.md + scripts/

storage/
├── doctor/global_lessons.json
└── patients/patient_001.json
Skillいつ触るファイル
global-lessonsPre-Consult / Reflectionglobal_lessons.json
patient-recordInterviewpatient_xxx.json

9-3. つまずきそうな点

  • ループ終了の判定（構造化出力が必要）
  • 教訓の肥大化（論理削除）
  • API コスト
