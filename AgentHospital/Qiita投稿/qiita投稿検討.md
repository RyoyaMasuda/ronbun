## 記事の位置づけ

| 項目 | 内容 |
|------|------|
| **ジャンル** | 論文解説 × 作ってみた × 現実適応の検討 |
| **主役** | 小さい Agent Hospital（ミニ版）の構築・実行ログ |

| **差別化** | 全文日本語訳 + Q&A + Strands 実装設計 + 再現実験 |

**避けること**

- 全文訳を Qiita にそのまま載せる（長すぎ・著作権リスク）
- 論文の言い換えだけで終わる（独自視点がない）

**全文日本語訳の扱い**

- `全文日本語訳/日本語訳_画像込み.html` は別リンク（GitHub 等）で公開
- Qiita 記事末尾の「参考」からリンク

## Part 2：ミニ版構築（`実装検討.txt` との対応）

### ワークフロー

```
Pre-Consult → Interview ⇄ Patient/Env → Reflection
```

| ノード | 種類 | 役割 |
|--------|------|------|
| Pre-Consult | Python 関数 | `read_global_lessons` で過去知見をロード |
| Interview | LLM エージェント | 問診・検査指示・診断 |
| Patient/Env | Python or LLM | 症状・検査結果を返す |
| Reflection | Python + LLM | 正解比較・反省・`update_global_lessons` |

### 記憶設計（エージェント型 RAG）

```
storage/
├── doctor/
│   └── global_lessons.json   # 共通ノート（教訓・反省）
└── patients/
    ├── patient_001.json      # 個別カルテ
    └── patient_002.json
```

- 完全上書きしない（Append-only + 論理削除 `is_active = False`）
- Naive RAG（ベクトル DB 一括検索）ではなく、LLM がツールで読み書き
